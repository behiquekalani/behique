Inside the guide

What you will actually read

On mindset

The audit produces decisions, not lists

Each system either stays as is, gets improved, gets replaced, or gets eliminated. Four options. For each system, you pick one and act on it within a specific time frame. Without a decision, an audit is just a documentation exercise.

On delivery

A delivery system that lives in your head is not a system

If the process cannot be followed by someone who has never met your client, it depends entirely on your memory. That is a single point of failure. Document the onboarding sequence. Document it now, before you need to hand it off.

On tools

Apply the replacement test to every tool

If this tool disappeared tomorrow, what would I use instead? If the answer is another tool you already pay for, the tool is redundant. If the answer is nothing, the tool is vestigial. Cut both categories. Keep only what you would rebuild.

On quarterly review

Avoidance is a reliable signal of system failure

When a necessary task keeps getting pushed, the usual assumption is that discipline is the problem. It is rarely the problem. The system for doing the task is unpleasant, unclear, or broken. Fix the system. The avoidance stops.
