What You Build

Six systems. One working setup.

Most people use Claude Code like a smarter terminal. These six configurations turn it into something that knows your project and holds context across sessions.

✎
CLAUDE.md file

The single file that tells Claude who you are, what project you are in, and how you want it to behave. Without this, Claude starts from zero every session. With it, it starts with full context.

🔗
Hooks system

Scripts that run automatically before and after Claude actions. Pre-commit security checks, post-edit linting, session summaries. The hooks run so you do not have to remember to.

⚙
Background agents

Claude running tasks while you work on something else. Long processes, batch operations, research loops. Configure it once and check back when it is done.

●
Build mode workflow

A structured way to work through a full build session with Claude. How to start it, how to maintain context mid-build, and how to hand off cleanly between sessions.

💾
Memory stack

A layered file system that keeps Claude oriented across sessions and across machines. Static rules, live state, and pattern logs. Nothing gets lost when you close the terminal.

▶
Session start protocol

The exact sequence Claude runs at the start of every session to load context, check current state, and open with something specific rather than a blank prompt.

No lost ideas

ADHD brains generate ideas faster than they can act on them. The session start protocol and hook system capture ideas immediately. Shower thought at 11pm. Voice note on the commute. It goes into the system before it evaporates.
