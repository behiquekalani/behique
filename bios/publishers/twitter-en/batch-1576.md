# Twitter Batch 1576

## Tweet 1
Hiring someone to do what a tool could handle is expensive.

Automating what needs judgment is worse.

Know which problem you actually have before you solve it.

## Tweet 2
If a task happens the same way every time, automate it before you hire for it.

Scheduling, invoicing, email sequences, social posting.

These are automation candidates. Build the system. Hire when the automation ceiling is hit.

## Tweet 3
You can't effectively delegate a task you've never done yourself.

Do it first. Document it. Then hand it off.

Skipping the first step leads to poor delegation and confused results.

## Tweet 4
Thread: how to decide whether to automate or delegate

Ask: is this task rule-based and repetitive?
Yes → automate first
No → does it require judgment and context?
Yes → document and delegate
No → ask why you're doing it at all

## Tweet 5
What's one task you do every week that follows the exact same steps every time?

That's your next automation project.

You should never manually do the same thing twice if a system can do it.
