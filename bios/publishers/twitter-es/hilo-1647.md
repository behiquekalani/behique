# Hilo 1647

## Tweet 1
Un sistema de referidos mal diseñado tiene una falla central: hace difícil para el cliente recomendar. El proceso de referir tiene que ser más simple que comprar.

## Tweet 2
El mejor referido no viene de un incentivo económico. Viene de un cliente que quedó tan bien servido que le duele no contarle a alguien.

## Tweet 3
¿Tus clientes actuales saben que agradeces las referencias? No lo asumas. Dilo. Pídelo explícitamente en el momento correcto del proceso.

## Tweet 4
El momento correcto para pedir un referido: justo después de que el cliente expresa satisfacción. No antes. No semanas después. En ese momento.

## Tweet 5
Diseña un mecanismo simple: un mensaje de texto que puedan reenviar, una página de referidos, un código. Simple. Claro. Accionable. Ese es el sistema.
