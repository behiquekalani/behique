Lesson 01

Setup and First Session

Install Claude Code, configure your API key, create your first CLAUDE.md file, and understand what Claude Code actually is (and isn't).

What Claude Code Is

Claude Code is a command-line tool that gives Claude direct access to your file system. It can read files, write files, run commands, and build entire projects. You talk to it in plain English. It writes the code. You review and approve.

Installation Steps
  1. Open Terminal (Mac) or your command line
  2. Run npm install -g @anthropic-ai/claude-code
  3. Navigate to your project folder: cd ~/your-project
  4. Run claude to start your first session
  5. It will ask for your Anthropic API key on first launch
Your First Prompt
Create a simple HTML page with a dark background, white text, and a centered heading that says "Hello World". Use -apple-system font. Make it look clean and minimal.

Lesson 02

The Memory Stack

Claude Code forgets everything between sessions. The Memory Stack is a system of markdown files that makes Claude remember your projects, preferences, tone, and context every time you start a new conversation.

The Three Files

CLAUDE.md

Static rules. Who you are. How Claude should talk to you. Your projects. Your tech stack. Your preferences. This file rarely changes.

primer.md

Dynamic state. What you're working on right now. What got done last session. Next steps. Blockers. Rewritten at the end of every session.

context.md

Big picture. Your goals. Your vision. Why you're building what you're building. Semi-static. Updated when your direction changes.

Prompt: Generate Your CLAUDE.md
Create a CLAUDE.md file in the root of this project. Include: - My name: _______________ - What I'm building: _______________ - My tech comfort level: _______________ - How I want you to communicate: direct, no fluff - My active projects (list them) - Rules: always read primer.md at session start, update it at session end
Prompt: Generate Your primer.md
Create primer.md with these sections: - Current Focus: (what I'm working on today) - Last Session: (what got done) - Next Steps: (concrete next actions) - Blockers: (anything stuck) - Open Loops: (things started but not finished)
How the Stack Works Together

When you start a new session, Claude automatically reads CLAUDE.md from your project root. Your CLAUDE.md tells Claude to also read primer.md and context.md. This means every session starts with full context of who you are, what you're building, and where you left off.

Pro Tip

Add this rule to your CLAUDE.md: "At the end of every session, rewrite primer.md completely with current state." This creates an automatic handoff between sessions. You never have to remember where you left off.

Lesson 04

PDF Generation Pipeline

Turn your HTML products into print-ready PDFs using headless browsers. Dark themes, proper margins, no headers or footers. The exact pipeline used to generate 100+ products.

The Command
"/Applications/Brave Browser.app/Contents/MacOS/Brave Browser" \ --headless \ --disable-gpu \ --print-to-pdf="/path/to/output.pdf" \ --no-margins \ --no-pdf-header-footer \ "file:///path/to/input.html"

This uses Brave (or Chrome) in headless mode to render the HTML and print it to PDF. No margins means your dark background goes edge to edge. No header/footer removes the default URL and page numbers.

Lesson 05

Batch Operations

How to make Claude process 10, 50, or 100 files at once. Generate thumbnails, convert PDFs, rename files, update content across an entire product catalog. The real productivity multiplier.

The Pattern

You show Claude one example of what you want. Then you tell it to repeat that across all files in a folder. It writes a script, you approve it, and it processes everything. 100 files in the time it takes to run one command.

Prompt: Batch PDF Generation
Write a bash script that converts every HTML file in /path/to/html-files/ to PDF using this command: "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser" \ --headless --disable-gpu \ --print-to-pdf="OUTPUT_PATH" \ --no-margins --no-pdf-header-footer \ "file://INPUT_PATH" Save PDFs to /path/to/pdf-output/ with the same filename but .pdf extension.
Prompt: Batch Thumbnail Generation
I have 20 product HTML files. For each one, create a 600x600 thumbnail HTML file with: - Dark background (#0a0a0f) - Product name in white, 42px, thin weight - Cyan line divider - "BEHIKE" at the bottom Then convert each to PNG using headless browser with --window-size=600,600 --force-device-scale-factor=2
Types of Batch Operations

File Operations

  • Rename files with patterns
  • Move files to organized folders
  • Convert formats (HTML to PDF)
  • Generate thumbnails at scale

Content Operations

  • Update branding across all files
  • Add print CSS to all HTML files
  • Generate Gumroad listings
  • Create product bundles
Pro Tip

Always do a dry run first. Ask Claude to generate the script and print what it would do without actually doing it. Review the output. Then run it for real. One bad script can rename 100 files incorrectly.

Version control your projects, push to GitHub, and deploy live to the internet. Even if you've never used Git before, Claude Code handles it for you.

Git Basics Through Claude

Git tracks every change to your files. GitHub stores your code online. Together they give you version history, backup, and the ability to deploy from anywhere. You don't need to memorize commands. Claude runs them for you.

GitHub Pages

Free static hosting. Perfect for HTML apps, portfolios, landing pages. Push to a repo, enable Pages, done.

Lesson 10

The Builder's Workflow

How to structure your daily workflow with Claude Code. Session management, when to use agents, how to review code, common mistakes, and the mindset that makes this work.

The Daily Session Structure
  1. Open terminal, navigate to project folder
  2. Run claude -- it reads your CLAUDE.md and primer.md
  3. State your goal for the session in one sentence
  4. Work in focused blocks (build, test, refine)
  5. At the end, tell Claude to update primer.md
  6. Review what got done. Plan tomorrow's session.
When to Use Different Modes

Interactive Mode

Default. You talk, Claude acts, you review. Best for new features, complex logic, anything you want to understand. You maintain control of every file change.

Agent / Autonomous Mode

Claude works independently with less oversight. Best for repetitive tasks, batch operations, or well-defined work where you trust the output pattern.

How to Review Code

You don't need to read every line. Focus on:

Always Check

  • Does it do what you asked?
  • Does it work when you test it?
  • Are file paths correct?
  • Are there hardcoded secrets?

Skip Unless Complex

  • Variable naming conventions
  • Code style preferences
  • Internal function structure
  • Optimization patterns
The Builder's Mindset

You Are the Architect

You decide what gets built, how it looks, who it's for, and when it ships. Claude is the contractor. You don't need to know how to pour concrete to design a building.

Ship Over Perfect

A shipped product beats a perfect draft. Version 1 doesn't need to be amazing. It needs to exist. You can always improve it in version 2. The hardest part is the first publish.

Prompt: Session Start Template
Read primer.md. Here's what I want to do today: [one sentence describing your goal] Let's start with the first step.
Prompt: Session End Template
Update primer.md with: - What we built today - What's working - What still needs to be done - Any blockers or open questions - The single most important next step
Common Mistakes to Avoid

Starting without context

Every session without CLAUDE.md is a session where Claude doesn't know who you are or what you're building. Set up the memory stack first.

Mega-prompts

A 500-word prompt asking for 10 things at once produces worse results than 10 focused prompts. Break complex work into steps.

Never testing

AI-generated code can look perfect and be broken. Always test in the browser, run the script, or try the feature before moving on.

Approval fatigue

After 50 approvals you start auto-accepting everything. That's when bugs slip in. Stay engaged. If you're tired, end the session.
