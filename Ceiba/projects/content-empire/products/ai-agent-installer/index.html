Run AI Locally with Ollama

▼

Ollama lets you run powerful AI models on your own computer. No cloud, no subscriptions, total privacy. It works on Mac, Windows, and Linux.

1. Install Ollama

curl -fsSL https://ollama.com/install.sh | sh

You should see: "Ollama has been installed successfully."

2. Download your first model

ollama pull llama3

This downloads the Llama 3 model. It may take a few minutes depending on your connection.

3. Start a conversation

ollama run llama3

You will see a prompt where you can type messages and get responses. Type /bye to exit.

Troubleshooting: If the install command fails, visit ollama.com/download and use the graphical installer instead. Make sure you have at least 8GB of RAM available.

Automate Workflows with n8n

▼

n8n is an open-source automation tool. Think of it like Zapier, but you own it. Connect AI to your email, calendar, database, and more.

1. Install n8n

npm install -g n8n

If you do not have npm, install Node.js first from nodejs.org.

2. Launch n8n

n8n start

Open your browser to http://localhost:5678. You will see the n8n editor.

3. Create your first workflow

Click "Add workflow," then add a Webhook trigger and an AI node. Connect them, activate, and you have a working AI endpoint.

Troubleshooting: If port 5678 is busy, run N8N_PORT=5679 n8n start instead. For persistent setup, use Docker: docker run -d -p 5678:5678 n8nio/n8n.

Build a Telegram Bot

▼

A Telegram bot gives you a personal AI assistant in your pocket. Send it messages and get intelligent replies, anywhere.

1. Create your bot

Open Telegram and message @BotFather. Send /newbot and follow the steps. Save the token it gives you.

2. Install the library

pip install python-telegram-bot openai

3. Create your bot file

from telegram.ext import ApplicationBuilder, MessageHandler, filters
import openai, os

client = openai.OpenAI(api_key=os.environ["OPENAI_API_KEY"])

async def reply(update, context):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role":"user","content":update.message.text}]
    )
    await update.message.reply_text(response.choices[0].message.content)

app = ApplicationBuilder().token(os.environ["BOT_TOKEN"]).build()
app.add_handler(MessageHandler(filters.TEXT, reply))
app.run_polling()

4. Run it

BOT_TOKEN="your-token" OPENAI_API_KEY="your-key" python bot.py

Your bot is now live. Message it on Telegram and it will respond with AI.

Troubleshooting: If you get a token error, double-check the token from BotFather. Make sure there are no extra spaces. If the bot does not respond, check that your OpenAI API key is valid and has credits.

Configuration Template

# agent-config.yaml
agent:
  name: "my-agent"
  version: "1.0.0"

llm:
  provider: "ollama"           # ollama | openai | anthropic
  model: "llama3"
  temperature: 0.7
  max_tokens: 4096

memory:
  backend: "chromadb"
  collection: "agent-memory"
  persist_directory: "./data/memory"

tools:
  - name: "web_search"
    enabled: true
  - name: "code_executor"
    enabled: true
    sandbox: true
  - name: "file_manager"
    enabled: true
    root_dir: "./workspace"

integrations:
  telegram:
    enabled: true
    token: "${BOT_TOKEN}"
  email:
    enabled: false
    smtp_host: "smtp.gmail.com"
    smtp_port: 587

deployment:
  platform: "docker"           # docker | railway | bare-metal
  port: 8080
  health_check: "/health"
  auto_restart: true

Quick Deploy Script

#!/bin/bash
# deploy-agent.sh
set -e

echo "Setting up AI Agent..."

# 1. Clone the repo
git clone https://github.com/behike/agent-starter.git
cd agent-starter

# 2. Create environment
python -m venv .venv
source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Copy and configure
cp agent-config.example.yaml agent-config.yaml
echo "Edit agent-config.yaml with your settings."

# 5. Launch
python main.py --config agent-config.yaml
