## SPEC 5: Niche Sniper (Remix 4)

### User Flow

1. **Daily scan (automated, 6 AM):**
   - Hutia runs the Reddit crawler across 15 subreddits
   - Searches for demand signals: "wish there was", "looking for", "no Mac version", "alternative to", "anyone know a tool for", "I would pay for"
   - Each result scored: (upvotes x 2) + (comments x 3) + recency_bonus
   - Results with score > 50 are flagged as opportunities

2. **Opportunity analysis (automated, 7 AM):**
   - For each flagged opportunity, the system runs two analyses:
   - **Physical product check:** Product research engine queries eBay/Amazon. Is this a physical product? What is the supply? What are margins? Score 0-100.
   - **Digital product check:** Is this a software gap? An education gap? A template gap? Score 0-100 based on complaint volume and solution complexity.

3. **Daily digest (8 AM, via Telegram):**
   - "/opportunities" shows today's top 5
   - Each opportunity shows: source (Reddit post link), demand score, product type (physical/digital), estimated effort, margin potential
   - Kalani reviews, selects actionable ones

4. **Action pipeline (semi-automated):**
   - For physical products: eBay listing agent generates a listing draft. Kalani reviews and publishes.
   - For digital products: System generates a product outline (title, chapters, price point, target audience). Stored in IDEAS_BACKLOG.md for future build sessions.
   - For software gaps: System generates a brief (problem statement, proposed solution, tech stack, estimated build time). Stored in IDEAS_BACKLOG.md.

5. **Feedback loop:**
   - Products that sell well get flagged for deeper niche exploration
   - Subreddits that produce the most opportunities get added to the priority scan list
   - Products that do not sell get their keywords removed from future scans
